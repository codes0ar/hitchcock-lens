## 1. 测试概述

### 1.1 APP功能架构

```
┌─────────────────────────────────────────────────────────────┐
│                      用户界面层 (UI Layer)                    │
│  ┌──────────┐ ┌──────────────┐ ┌──────────┐ ┌────────────┐  │
│  │ 录制按钮  │ │ 人脸锁定指示器 │ │ Zoom显示  │ │  设置面板   │  │
│  └──────────┘ └──────────────┘ └──────────┘ └────────────┘  │
├─────────────────────────────────────────────────────────────┤
│                    控制逻辑层 (Control Layer)                 │
│  ┌──────────────────┐    ┌──────────────────────────────┐   │
│  │  比例控制算法      │    │  EMA平滑 (SMOOTHING=0.15)   │   │
│  │  error = target   │ →  │  zoom += 0.15 * error       │   │
│  │       - current   │    │                              │   │
│  └──────────────────┘    └──────────────────────────────┘   │
├─────────────────────────────────────────────────────────────┤
│                   算法层 (Algorithm Layer)                    │

│  │           expo-face-detector (accurate模式)          │    │
│  │  输入: 摄像头帧 → 输出: 人脸边界框 (x,y,width,height) │    │
│  └─────────────────────────────────────────────────────┘    │
├─────────────────────────────────────────────────────────────┤
│                   硬件抽象层 (Hardware Layer)                  │
│  ┌──────────────┐  ┌──────────────┐  ┌──────────────────┐   │

│  │  (expo-camera)│  │              │  │  (expo-media-lib)│   │
│  └──────────────┘  └──────────────┘  └──────────────────┘   │
└─────────────────────────────────────────────────────────────┘
```

### 2.1 人脸检测与锁定测试

#### TC-FD-001: 首次人脸检测与目标锁定
| 项目 | 内容 |
|------|------|
| **用例ID** | TC-FD-001 |
| **优先级** | P0 |
| **测试目的** | 验证首次检测到人脸时，APP能正确锁定人脸像素大小作为控制目标 |
| **前置条件** | APP已安装并启动，摄像头权限已授权，预览界面正常显示 |
| **测试步骤** | 1. 打开APP，进入拍摄界面<br>2. 将人脸置于取景框中央（距离约2米）<br>3. 等待人脸被检测到<br>4. 记录锁定的人脸像素大小 |
| **期望结果** | 1. 人脸检测框正确显示在人脸位置<br>2. 人脸锁定指示器从"搜索中"变为"已锁定"<br>3. 锁定的人脸像素宽度值合理（预期80-300像素范围，取决于距离）<br>4. 该值被记录为控制目标值 |
| **通过标准** | 人脸检测成功且锁定指示器状态切换时间 ≤ 500ms；锁定值与实际人脸像素宽度误差 ≤ 5% |

### 4.2 人脸大小稳定性测试

#### TC-ACC-001: 静态场景人脸大小稳定性
| 项目 | 内容 |
|------|------|
| **用例ID** | TC-ACC-001 |
| **优先级** | P0 |
| **测试目的** | 测量固定场景下人脸像素大小的稳定性（无运动时应无变化） |
| **测试设备** | 三脚架固定手机，被摄体固定位置 |
| **测试步骤** | 1. 手机固定在三脚架上<br>2. 被摄体坐在固定位置（距离2m）<br>3. 锁定人脸<br>4. 录制30秒视频<br>5. 逐帧提取并测量人脸像素宽度<br>6. 计算统计指标 |
| **测量指标** | - 人脸像素宽度均值 (μ)<br>- 人脸像素宽度标准差 (σ)<br>- 变异系数 CV = σ/μ × 100%<br>- 最大绝对偏差 = max\|width - μ\| |
| **通过标准** | **CV < 3%**（理想）; CV < 5%（可接受）; 最大绝对偏差 < 10% μ |
| **计算公式** | $$CV = \frac{\sigma}{\mu} \times 100\% = \frac{\sqrt{\frac{1}{N}\sum_{i=1}^{N}(w_i - \mu)^2}}{\mu} \times 100\%$$ |

### 4.3 Zoom响应精度测试

#### TC-ACC-005: 距离-Zoom补偿精度
| 项目 | 内容 |
|------|------|
| **用例ID** | TC-ACC-005 |
| **优先级** | P0 |
| **测试目的** | 验证zoom调整是否精确补偿距离变化 |
| **测试原理** | 根据薄透镜公式，人脸像素大小与 (zoom / distance) 成正比。当zoom正确补偿距离变化时，人脸像素大小应保持恒定。 |
| **测试步骤** | 1. 在已知距离序列（1m, 1.5m, 2m, 3m, 4m, 5m）分别锁定人脸<br>2. 记录各距离的初始人脸大小和zoom值<br>3. 从基准距离移动到新距离<br>4. 测量新距离下的人脸大小偏差<br>5. 计算补偿误差 |
| **测量指标** | 补偿误差 = \|实际人脸大小 - 目标人脸大小\| / 目标人脸大小 × 100% |
| **通过标准** | **补偿误差 < 10%**（各距离点）; 平均补偿误差 < 5% |

#### TC-ACC-006: Zoom线性响应测试
| 项目 | 内容 |
|------|------|
| **用例ID** | TC-ACC-006 |
| **优先级** | P1 |

| **通过标准** | 实测曲线与理论曲线相关系数 R² > 0.95; 残差标准差 < 5% |

#### 7.2.3 自动化数据收集脚本

```python
# zoom_analysis.py - 精度测试数据分析脚本示例
import cv2
import pandas as pd
import numpy as np
from pathlib import Path

def extract_face_sizes(video_path, output_csv):
    """从视频中提取每帧的人脸像素大小"""
    cap = cv2.VideoCapture(video_path)
    face_cascade = cv2.CascadeClassifier('haarcascade_frontalface_default.xml')
    
    data = []
    frame_idx = 0
    
    while True:
        ret, frame = cap.read()
        if not ret:
            break
            
        gray = cv2.cvtColor(frame, cv2.COLOR_BGR2GRAY)
        faces = face_cascade.detectMultiScale(gray, 1.1, 4)
        
        if len(faces) > 0:
            # 取最大人脸
            x, y, w, h = max(faces, key=lambda f: f[2] * f[3])
            data.append({
                'frame': frame_idx,
                'face_width': w,
                'face_height': h,
                'face_area': w * h,
                'timestamp_ms': cap.get(cv2.CAP_PROP_POS_MSEC)
            })
        
        frame_idx += 1
    
    cap.release()
    
    df = pd.DataFrame(data)
    df.to_csv(output_csv, index=False)
    
    # 计算精度指标
    target_size = df['face_width'].iloc[0]  # 第一帧作为目标
    mean_size = df['face_width'].mean()
    std_size = df['face_width'].std()
    cv = (std_size / mean_size) * 100
    max_error = ((df['face_width'] - target_size).abs() / target_size * 100).max()
    
    print(f"=== 精度分析结果 ===")
    print(f"目标人脸宽度: {target_size:.1f} px")
    print(f"平均人脸宽度: {mean_size:.1f} px")
    print(f"标准差: {std_size:.2f} px")
    print(f"变异系数 CV: {cv:.2f}%")
    print(f"最大偏差: {max_error:.2f}%")
    print(f"结果: {'PASS' if cv < 5 else 'FAIL'}")
    
    return df, cv

def analyze_zoom_smoothness(zoom_log_path):
    """分析zoom值的平滑度"""
    df = pd.read_csv(zoom_log_path)  # columns: timestamp, zoom_value
    
    # 计算变化率
    df['zoom_rate'] = df['zoom_value'].diff() / df['timestamp'].diff()
    
    # 方差
    rate_variance = df['zoom_rate'].var()
    
    # FFT分析高频能量
    fft = np.fft.fft(df['zoom_value'].fillna(method='ffill'))
    freqs = np.fft.fftfreq(len(df))
    energy_ratio = np.sum(np.abs(fft[np.abs(freqs) > 0.05])) / np.sum(np.abs(fft))
    
    print(f"=== 平滑度分析 ===")
    print(f"变化率方差: {rate_variance:.6f}")
    print(f"高频能量比: {energy_ratio*100:.2f}%")
    print(f"结果: {'PASS' if energy_ratio < 0.05 else 'FAIL'}")
    
    return rate_variance, energy_ratio
```

| 往返运动 | 1m→3m | 0.5m/s | < 8% | < 10% | ≥ 4.0分 |
| 阶梯距离 | 1-5m | 阶梯变化 | < 5% | < 10% | ≥ 4.0分 |

## 10. 风险评估

### 10.1 高风险测试项

| 排名 | 风险项 | 风险等级 | 影响 | 根因分析 | 缓解策略 |
|------|--------|---------|------|---------|---------|
| 1 | **人脸检测在低光/逆光下的精度** | 🔴 高 | 精度测试在暗光场景可能大面积失败，影响用户体验 | expo-face-detector accurate模式依赖设备ISP处理后的图像，暗光下噪点大 | 1. 定义明确的低光降级策略<br>2. 增加UI提示引导用户改善光线<br>3. 考虑集成第三方增强型人脸检测SDK作为备选 |
| 2 | **Android设备碎片化兼容性** | 🔴 高 | 不同厂商的摄像头API实现差异大，zoom控制可能不一致 | Camera2 API厂商定制化；Expo统一抽象层可能无法覆盖所有差异 | 1. 建立设备分级策略<br>2. 核心设备全覆盖测试<br>3. 云测平台覆盖长尾设备<br>4. 收集线上数据持续优化 |
| 3 | **EMA平滑参数适配** | 🟠 中 | SMOOTHING_FACTOR=0.15可能在某些场景下过于保守或激进 | 固定参数无法适应所有运动速度 | 1. 测试不同参数的效果矩阵<br>2. 考虑自适应平滑策略<br>3. 提供用户可调选项 |
| 4 | **长时间录制稳定性** | 🟠 中 | 15分钟持续运行可能暴露内存泄漏或热节流 | React Native桥接通信、视频编码器资源管理 | 1. 强化的长时间压力测试<br>2. 内存使用监控告警<br>3. 定期释放缓存策略 |
| 5 | **多摄像头切换连续性** | 🟠 中 | 多摄设备在zoom跨越镜头焦段时可能产生画面跳变 | 镜头物理切换需要时间，曝光/白平衡可能变化 | 1. 测试各镜头切换点的用户体验<br>2. 在切换点附近限制zoom范围<br>3. 添加过渡动画 |
| 6 | **快速运动场景精度** | 🟡 低 | 高速运动下系统延迟导致补偿滞后 | 算法固有延迟（检测+控制环路） | 1. 明确性能上限<br>2. 用户教育（推荐运动速度）<br>3. 考虑预测控制算法 |

### 10.2 测试阶段风险

| 风险 | 可能性 | 影响 | 应对措施 |
|------|--------|------|---------|
| 测试设备不足 | 中 | 兼容性覆盖不全 | 租用云测设备（AWS Device Farm / Firebase Test Lab） |
| 测试环境搭建复杂 | 中 | 精度测试延迟 | 简化初始环境，用人工标记代替轨道；逐步完善 |
| 自动化脚本开发时间 | 高 | 测试效率低 | 优先实现核心数据采集自动化；UI自动化使用Maestro快速搭建 |
| 人员技能要求 | 低 | 视频分析能力不足 | 提供OpenCV培训；使用现成分析脚本 |

### 10.3 推荐测试优先级

```

└── 用户体验: 基础场景

第3周 (P1+P2):
├── 兼容性: P1设备
├── 用户体验: 特殊场景 + 调查问卷

└── 报告: 汇总分析
```
